Git command

Git Version 2.x

  • git 初始化目錄

      git init

    目錄底下會出現.git資料夾，目的是要讓 Git 開始對這個目錄進行版本控制。

  • 將檔案加入索引

      git add . `加入所有檔案的變化`
  git add -u `加入Modified & deleted 不包含new`

  • 加入版本資訊

      git commit

    可以在後面加入-m ，每一段的 message 都會被一個空行隔開 git commit -m text1 -m text2

  • 設定遠端節點(Github)

      git remote add origin git@github.com:yourname/testPractice-git.git

    add 指令是指要加入一個遠端的節點
    在這裡的 origin 指伺服器的位置 (這裡用Github為例)
    可以在資料夾 .git/config看到相關的設定

  • 設定好遠端節點，接下來可以把目錄的檔案推上去

      git push -u origin master

這裡是將local master branch 推到 origin上，如果remote 沒有master就會自動創建

  • 查看當前狀態

      git status

  • discard change

    git checkout –file

  • git clean清除未加入版控的資料

SiteGround clone from github

因為沒有買 SiteGround GoGeek 方案
所以要進行版本控制得自己來

首先ssh 到 SiteGround 可以參考 SSH to SiteGround

登入後我們要對 SiteGround 創建public key

ssh-keygen -t rsa
Enter file in which to save the key (/Users/.ssh/id_rsa): 這裡可以打檔案名稱(ir_rsa_test)若空白則是預設id_rsa)
再來按Enter即可

然後複製id_rsa.pub到Github->Setting->SSH and GPG keys

接著回到 SiteGround public_html下跑

git init
vim ignore 加入排除的檔案
git add .
git remote add origin remote.git
git push -u origin master

若要clone下來的直接跑git clone repo就可以了

SSH to SiteGround

首先在本機端創一個New key

ssh-keygen -t rsa
Enter file in which to save the key (/Users/.ssh/id_rsa): 這裡可以打檔案名稱(ir_rsa_test)若空白則是預設id_rsa)
再來按Enter即可

複製剛剛創好的public key
貼到Siteground -> DEVS -> SSH Keys Manager->Import

完成面步驟即可

接下來到電腦上使用ssh來測試連接

ssh -i ~/.ssh/id_rsa siteground使用者名稱＠網站domain -p18765

使用者名稱在siteground創好key的地方點擊Actions->SSH Credentials 就可以看到

Magento 介紹

Magento 是使用PHP為基礎所開發的電子商務平台(購物車系統)

擁有強大的各種訂單、客戶、金流管理功能、且有彈性，讓客戶或網站操作管理員都有很棒的服務。

由於在台灣 Magento 的技術文件不怎麼多，在開發研究多要往國外的文件去尋找，要熟練這個 framework 需要不少的時間，所以需要一些客製化的功能可以去尋找一些額外的Extension免費或付費 Magento Marketplace 或者網路上Opensource(Github 等等…) 讓你可以快速的找到你要的功能，節省自己開發的時間或做為開發的參考。
